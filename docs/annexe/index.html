Principes d'architecture pour le cloud

Pour concevoir des applications faciles à exploiter et capable de passer à l'échelle voir :

  • 12factor.net - Les 12 facteurs
  • learn.microsoft.com - Modèles de conception de cloud dont :
    • learn.microsoft.com - Modèle Surveillance de point de terminaison
    • learn.microsoft.com - CQRS pour pouvoir s'adapter à la charge plus facilement sur la seule diffusion (dans notre exemple : /wms, /wfs,... vs /geoserver/)
    • learn.microsoft.com - Modèle Figuier étrangleur pour gérer la migration d'un ancien vers un nouveau service.
    • learn.microsoft.com - Modèle Nouvelle tentative et learn.microsoft.com - Modèle Disjoncteur pour survivre aux instabilités d'une plateforme d'hébergement ou d'un service tiers.
